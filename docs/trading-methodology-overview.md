**작성일**: 2026-03-02

## 1. 시스템 철학

### 1.1 왜 체계적 트레이딩인가

체계적(systematic) 트레이딩은 감정적 판단을 배제하고, 검증된 규칙에 기반하여 반복 가능한
투자 의사결정을 내리는 것을 목표로 한다. 본 시스템은 네 가지 핵심 축을 통합한다:

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│   ┌─────────────────┐    ┌─────────────────────────────┐    │
│   │  정량적 스코어링  │    │  검증된 방법론 인코딩        │    │
│   │  Quantitative    │◄──►│  Verified Methodology        │    │
│   │  Scoring         │    │  Encoding                    │    │
│   └────────┬────────┘    └──────────────┬──────────────┘    │
│            │                             │                   │
│            ▼                             ▼                   │
│   ┌─────────────────┐    ┌─────────────────────────────┐    │
│   │  스킬 체이닝     │    │  자동 자기 개선               │    │
│   │  Skill           │◄──►│  Automatic Self-             │    │
│   │  Chaining        │    │  Improvement                 │    │
│   └─────────────────┘    └─────────────────────────────┘    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 단타를 지양하는 이유

| 요인 | 단타 (데이 트레이딩) | 중장기 (스윙/포지션) |
|------|---------------------|---------------------|
| 거래비용 | 연간 수백~수천 회 거래 → 비용 누적 | 연간 수십 회 → 비용 최소 |
| 세금 효율 | 단기 양도세 (높은 세율) | 장기 양도세 (낮은 세율) |
| 정보 우위 | HFT/기관과 경쟁 → 열위 | 펀더멘탈 분석으로 우위 가능 |
| 오버피팅 리스크 | 노이즈에 과적합 위험 높음 | 구조적 팩터에 기반 → 안정적 |
| 심리적 부담 | 지속적 모니터링, 번아웃 | 주/월 단위 관리 → 지속 가능 |
| 학술적 검증 | 개인 투자자 수익률 저조 입증 | 팩터 프리미엄 장기 검증됨 |

## 2. 4대 핵심 축 개요

### 2.1 정량적 스코어링 (Quantitative Scoring)

> 상세: [`quantitative-scoring-methodologies.md`](quantitative-scoring-methodologies.md)

**목적**: 모든 투자 대상을 객관적 수치로 평가하여 주관적 편향 제거

**핵심 구조**:
```
LAYER 1: 안전성 필터 (Pass/Fail)
  - Altman Z-Score > 1.81 (파산 리스크 배제)
  - Beneish M-Score < -1.78 (분식회계 배제)
  - 최소 유동성 충족

LAYER 2: 멀티팩터 복합 점수 (0-100)
  기본적 분석 (40-50%):
    - 가치: P/E, EV/EBIT, Earnings Yield
    - 품질: ROE, ROIC, Piotroski F-Score
    - 성장: EPS/매출 CAGR
  기술적 분석 (25-35%):
    - 추세: 200일 MA 대비 위치
    - 모멘텀: 12-1개월 수익률
    - 거래량: OBV, 수급 확인
  센티먼트 (15-25%):
    - 애널리스트 추정치 변경
    - 내부자 거래
    - 공매도 비율 변화

LAYER 3: 리스크 조정
  - 변동성/낙폭/테일 리스크 패널티

LAYER 4: 포지션 사이징
  - Fractional Kelly (1/4~1/2)
  - 변동성 기반 등위험 배분
```

**검증된 스코어링 모델**:

| 모델 | 용도 | 점수 범위 | 핵심 지표 |
|------|------|---------|---------|
| Piotroski F-Score | 가치주 품질 평가 | 0-9 | 수익성, 재무건전성, 운영효율 |
| Altman Z-Score | 파산 리스크 필터 | 연속값 | 유동성, 수익성, 부채, 매출 |
| Beneish M-Score | 분식회계 탐지 | 연속값 | 매출채권, 자산, 감가상각 비율 |
| Mohanram G-Score | 성장주 품질 평가 | 0-8 | 수익성, 안정성, 보수성 |

### 2.2 스킬 체이닝 (Skill Chaining)

> 상세: [`skill_chaining_and_self_improvement_research.md`](skill_chaining_and_self_improvement_research.md)

**목적**: 개별 트레이딩 기술(skill)을 파이프라인으로 연결하여 시너지 극대화

**기원**: 계층적 강화학습(Hierarchical Reinforcement Learning)의 스킬 체이닝 개념
- Konidaris & Barto: 복잡한 장기 과제를 연속된 하위 과제로 분해
- NeurIPS 2024 SCaR: 양방향 정렬(bi-directional alignment) → 인접 스킬 간 원활한 연결

**9계층 트레이딩 스킬 체인**:
```
Layer 1: 데이터 인프라        → 클린 데이터, 피처 스토어
Layer 2: 레짐 감지           → HMM, 2-4개 시장 상태 분류
Layer 3: 시그널 생성          → 다중 전략 (트렌드, 평균회귀, 팩터)
Layer 4: 전략 선택/메타러닝    → XGBoost 메타러너, 전략 가중치
Layer 5: 포지션 사이징        → Fractional Kelly, 변동성 타겟팅
Layer 6: 리스크 관리          → ATR 스탑, 낙폭 제어, 집중도 한도
Layer 7: 실행                → 주문 생성, TWAP/VWAP
Layer 8: 성과 분석           → 트레이드/전략/팩터/스킬별 어트리뷰션
Layer 9: 자기 개선            → Walk-Forward 재최적화, 모델 재학습
```

**레짐 감지** (핵심 스킬):
- Hidden Markov Model이 골드 스탠더드
- 입력: 로그 수익률, 실현 변동성, VIX, 수익률 곡선
- 출력: 레짐 라벨 + 확률 (저변동 상승, 고변동 상승, 저변동 횡보, 고변동 하락)
- 각 레짐에 맞는 전문가 전략 배정

### 2.3 자동 자기 개선 (Automatic Self-Improvement)

> 상세: [`skill_chaining_and_self_improvement_research.md`](skill_chaining_and_self_improvement_research.md) §4-§8

**목적**: 시스템이 스스로의 성과를 학습하여 지속적으로 개선

**핵심 메커니즘**:

```
┌─────────────┐   ┌──────────────────┐   ┌─────────────────┐
│  라이브      │──►│  성과 모니터링    │──►│  성과 어트리뷰션  │
│  트레이딩    │   │                  │   │  (4단계)        │
└─────────────┘   └──────────────────┘   └────────┬────────┘
      ▲                                           │
      │                                           ▼
┌─────┴───────┐   ┌──────────────────┐   ┌─────────────────┐
│  업데이트    │◄──│  검증 게이트     │◄──│  모델 재학습     │
│  배포        │   │  (WFA 통과 시)   │   │  파라미터 최적화  │
└─────────────┘   └──────────────────┘   └─────────────────┘
```

**Walk-Forward Optimization 설정 (포지션 트레이딩용)**:

| 파라미터 | 권장 값 | 근거 |
|---------|--------|------|
| In-Sample 윈도우 | 2-5년 | 복수 시장 사이클 포착 |
| Out-of-Sample 윈도우 | 6-12개월 | 보유 기간에 부합 |
| 롤 스텝 | 3-6개월 | 신선함과 안정성 균형 |
| 윈도우 유형 | 앵커드 (확장형) | 장기 전략에 안정적 |
| 적합도 함수 | 샤프비율 / 소르티노 | 리스크 조정 기준 |
| WFE 목표 | > 50% | OOS/IS 비율 |

**오버피팅 탐지**:
- PBO (Probability of Backtest Overfitting) < 10%
- Deflated Sharpe Ratio: 시행 횟수 보정
- 파라미터 안정성: ±10% 변경 시 성과 유지

**주기적 개선 스케줄**:

| 주기 | 작업 | 비고 |
|------|------|------|
| 일간 | 레짐 감지 업데이트, 리스크 모니터링 | HMM 최신 데이터 반영 |
| 주간 | 전략 가중치 업데이트, 포지션 조정 | 메타러너 재평가 |
| 월간 | Walk-Forward 재최적화, 깊이 있는 성과 분석 | 핵심 파라미터 조정 |
| 분기 | 전체 ML 모델 재학습 | 최신 데이터로 재학습 |
| 연간 | 전략 유효성 종합 검토 | 구조적 변화 반영 |

### 2.4 검증된 방법론 인코딩 (Verified Methodology Encoding)

> 상세: [`verified-methodologies-and-risk-management.md`](verified-methodologies-and-risk-management.md)

**목적**: 학술적/실전 검증된 방법론을 재현 가능한 규칙으로 변환

**검증된 방법론 비교**:

| 방법론 | 유형 | 보유 기간 | 리밸런싱 | 벤치마크 초과 |
|-------|------|---------|---------|-------------|
| Fama-French 팩터 | 팩터 틸트 | 수개월~수년 | 연간 | +2-5%/년 |
| CAN SLIM | 성장주 선별 | 수주~수개월 | 지속 모니터링 | ~28% CAGR (1998-2007) |
| Magic Formula | 가치+품질 | 1년 | 연간 | +3-18%/년 |
| Dual Momentum | 전술적 배분 | 1-12개월 | 월간 | +4.4%/년 (1974-2013) |
| 트렌드 팔로잉 | 추세 추종 | 수주~수개월 | 시그널 기반 | 낮은 수익, 낮은 낙폭 |
| Risk Parity | 전략적 배분 | 상시 | 연간 | ~7.3% CAGR, 1/4 낙폭 |

**인코딩 원칙**:
1. **명확한 규칙**: 모든 조건 수치화 (주관적 판단 배제)
2. **결정론적 실행**: 동일 데이터 → 동일 결정 → 재현 가능
3. **파라미터 최소화**: 전략당 2-4개 핵심 파라미터
4. **계층적 필터링**: 매크로 → 유니버스 → 스코어링 → 사이징 → 리스크

### 3.2 리스크 관리 계층

```
Level 1 - 거래 단위: ATR 스탑로스 (2.5-3.5x ATR)
                     자본의 1% 리스크/거래
    │
Level 2 - 전략 단위: 전략별 최대 낙폭 15%
                     연속 손실 제한
    │
Level 3 - 포트폴리오: 전체 낙폭 한도 20%
                      섹터 25%, 단일 종목 5-8%
    │
Level 4 - 시스템: 서킷 브레이커 (일일 3% 손실 → 거래 중단)
                  킬 스위치 (비상 전포지션 청산)
```

**낙폭 방어 3단계**:
- 10% 낙폭: 신규 진입 중단, 모니터링 강화
- 15% 낙폭: 포지션 50% 축소, 방어적 전환
- 20% 낙폭: 전체 청산, 최소 1개월 냉각기 후 25%씩 점진 재진입

## 4. 구현 로드맵

### Stage 1: 최소 실행 가능 시스템 (MVP) — 1-2개월

```
목표: 첫 번째 체계적 시스템 가동
복잡도: ★☆☆☆☆

매크로 필터:
  - Dual Momentum (SPY vs VXUS vs AGG)
  - 월 1회 체크, 자산군 전환

종목 선별:
  - Magic Formula (Earnings Yield + Return on Capital)
  - 상위 20-30 종목 등가중

리스크:
  - 최대 낙폭 20% → 전체 청산
  - 섹터 최대 25%

리밸런싱: 연 1회 (세금 최적화: 손실 11개월, 이익 13개월)

도구: 스프레드시트 or Python + yfinance
```

### Stage 2: 기본 정량 시스템 — 3-6개월

```
목표: 멀티팩터 스코어링 도입
복잡도: ★★☆☆☆

유니버스 필터:
  - Altman Z-Score > 1.81
  - Beneish M-Score < -1.78
  - 최소 유동성/시가총액

스코어링:
  - 가치 (25%): EV/EBIT, P/B
  - 품질 (25%): F-Score, ROE
  - 모멘텀 (25%): 12-1개월 수익률
  - 성장 (15%): EPS CAGR
  - 센티먼트 (10%): 추정치 변경

포지션 사이징: Fractional Kelly (1/4)
리스크: ATR 스탑 (3x ATR), 3단계 낙폭 방어
리밸런싱: 월/분기

도구: Python + Backtrader + yfinance/FMP
```

### Stage 3: 고급 적응형 시스템 — 6-12개월

```
목표: 레짐 감지 + 멀티 전략 앙상블
복잡도: ★★★★☆

레짐 감지: 2-3상태 HMM (hmmlearn)
전략: 트렌드 팔로잉 + 평균회귀 + 팩터 모델
전략 선택: 레짐 기반 규칙 → XGBoost 메타러너
```

## 5. 핵심 수식 레퍼런스

### 스코어링

```
복합 점수 = w_F * 기본적_점수 + w_T * 기술적_점수 + w_S * 센티먼트_점수

Piotroski F-Score = Σ(F1..F9), 각 항목 0 또는 1, 범위 0-9
Altman Z-Score = 1.2*A + 1.4*B + 3.3*C + 0.6*D + 1.0*E
Beneish M-Score = -4.84 + 0.92*DSRI + 0.528*GMI + ...
```

### 포지션 사이징

```
Kelly: f* = (μ - r) / σ²  (연속), f* = (bp - q) / b  (이산)
실전: position = min(Kelly × fraction, 리스크버짓, 변동성타겟, 최대한도)
ATR 사이징: position = (자본 × 리스크%) / (ATR × 승수)
```

### 리스크 측정

```
Sharpe = (R_p - R_f) / σ_p          (목표: > 1.0)
Sortino = (R_p - R_f) / σ_downside  (목표: > 1.5)
Calmar = (R_p - R_f) / MDD          (목표: > 0.5)
MDD = max(HWM_t - NAV_t) / HWM_t   (목표: < 20%)
```

### 팩터 모델

```
Fama-French 5F: R_i - R_f = α + β_M(R_M-R_f) + β_S·SMB + β_V·HML + β_P·RMW + β_I·CMA + ε
IR ≈ IC × √Breadth  (기본법칙: 예측력 × 독립적 베팅 수)
```
