# 자기 개선 규칙 상세

## 진단 임계값

| 진단 항목 | 임계값 | 상태 |
|----------|--------|------|
| 레짐 정확도 | < 55% | 개선 필요 |
| 시그널 IC | < 0.03 | 개선 필요 |
| Kelly 효율 | < 70% | 개선 필요 |
| 전략 상관관계 | > 0.7 | 분산 부족 |
| MDD | > 15% | 리스크 과다 |
| WFE | < 50% | 오버피팅 의심 |

## 처방 매핑

### 레짐 정확도 < 55%
- **즉시**: HMM 모델 피처 점검
- **중기**: 피처 추가 (신용 스프레드, 경기선행지수)
- **장기**: HMM → 딥러닝 전환 검토

### 시그널 IC < 0.03
- **즉시**: 피처 중요도 드리프트 체크
- **중기**: Walk-Forward 재최적화
- **장기**: 방법론 교체/추가 검토

### Kelly 효율 < 70%
- **즉시**: 변동성 추정 방법 점검
- **중기**: Kelly 파라미터 재보정 (1/4 → 1/5 고려)
- **장기**: Bayesian Kelly 도입

### 전략 상관관계 > 0.7
- **즉시**: 상관 높은 전략 쌍 식별
- **중기**: 전략 가중치 조정
- **장기**: 비상관 전략 추가 (Mean Reversion 등)

### MDD > 15%
- **즉시**: 스탑 타이트닝 (ATR 3.0x → 2.5x)
- **중기**: 레짐 필터 강화 (Bear 시 현금 비중 확대)
- **장기**: 포트폴리오 헤지 도입

### WFE < 50%
- **즉시**: 파라미터 수 축소
- **중기**: 정규화(L1/L2) 적용
- **장기**: 모델 단순화 (less is more)

## 개선 주기

| 시간축 | 주기 | 범위 |
|--------|------|------|
| 스윙 | 월간 | 시그널 가중치, 스탑, 사이징 |
| 포지션 | 분기 | 전략 구성, 레짐 모델, 리스크 한도 |
| 시스템 | 반기 | 전체 아키텍처, 데이터 소스, 브로커 |

## 절대 규칙

1. **자동 수정 금지**: 모든 변경은 제안 → 사용자 승인 → 적용
2. **A/B 테스트**: 변경 전 Paper Trading으로 최소 1개월 검증
3. **롤백 계획**: 모든 변경에 롤백 방법 명시
4. **변경 기록**: 모든 파라미터 변경 이력 보존
