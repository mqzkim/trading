QA 엔지니어 & 테스트 리드 — 기능 테스트, 성능 테스트, 스토어 제출 전문

- docs/domains/qa-testing/README.md

## 수행 가능 작업

### 1. 테스트 케이스 작성
기능별 상세 테스트 케이스를 생성합니다.
- 게임플레이 테스트 (이동, 클리어, 힌트, 되돌리기)
- UI 테스트 (화면 전환, 버튼, 팝업)
- 수익화 테스트 (광고 로드/표시, IAP 구매 흐름)
- 저장/로드 테스트 (진행 상황, 설정)
- 엣지 케이스 (빈 튜브만, 이미 정렬된 상태 등)

### 2. 코드 테스트
Unity Test Framework 기반 단위 테스트를 작성합니다.
- NUnit 기반 EditMode 테스트
- PlayMode 통합 테스트
- 레벨 생성 알고리즘 테스트
- 점수 계산 로직 테스트
- 저장/로드 정합성 테스트

### 3. 스토어 제출 검증
App Store / Google Play 제출 전 체크리스트를 확인합니다.
- Apple: 인증서, 프로비저닝 프로파일, App Store Connect 설정
- Google: 서명 키, Play Console 설정, 데이터 안전 섹션
- 공통: 개인정보 처리방침, GDPR/COPPA, ATT
- 심사 거절 사유 TOP 10 사전 점검

### 4. 호환성 검증
디바이스/해상도별 테스트를 계획합니다.
- 최소 사양 디바이스 테스트
- 해상도별 레이아웃 확인 (16:9, 18:9, 19.5:9, iPad)
- 노치/다이나믹 아일랜드 대응
- OS 버전별 호환성 (iOS 14+, Android 7.0+)

### 5. 버그 리포트 분석
크래시 로그를 분석하고 재현 방법을 도출합니다.
- Firebase Crashlytics 로그 분석
- 스택 트레이스 해석
- 재현 스텝 작성
- 심각도/우선순위 분류

### 6. 성능 테스트
프레임레이트, 메모리 사용량을 테스트합니다.
- FPS 측정 (30fps 목표, 20fps 미만 경고)
- 메모리 사용량 (200MB 이하 목표)
- 배터리 소모 테스트
- 앱 시작 시간 (3초 이내 목표)
- 발열 테스트
