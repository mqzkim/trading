# /simplify

최근 변경된 코드를 분석하여 불필요한 복잡성을 찾아내고 단순화를 제안한다.

## 사용법
```
/simplify              # 최근 커밋 변경사항 분석
/simplify <file>       # 특정 파일 분석
/simplify --all        # 스테이징 + 언스테이징 전부 분석
```

## 분석 항목

1. **중복 코드** — 3줄 이상 반복되는 패턴
2. **과도한 추상화** — 한 번만 사용되는 헬퍼/유틸리티
3. **사용하지 않는 코드** — import했지만 사용 안 하는 모듈, 호출되지 않는 함수
4. **불필요한 복잡성** — 단순한 로직의 과도한 분기, 불필요한 제네릭
5. **Boris 3줄 원칙** — "비슷한 3줄이 조기 추상화보다 낫다"

## 출력 형식

```
[simplify] src/auth/service.ts
- L23-45: fetchUser()와 getUser()가 거의 동일 → 하나로 통합 가능
- L67: UserMapper 클래스는 한 번만 사용됨 → 인라인 가능
- L89: import { unused } from './utils' → 미사용 import 제거

제안된 변경: 3건 | 예상 라인 감소: -28줄
```
