    - StopHitAlert, TargetReachedAlert domain events added to execution events

  <action>
    1. Create `src/execution/application/` directory with `__init__.py`.

    2. Create `src/execution/application/commands.py`:
       ```python
       @dataclass
       class GenerateTradePlanCommand:
           symbol: str
           entry_price: float
           atr: float
           capital: float
           peak_value: float
           current_value: float
           intrinsic_value: float
           composite_score: float
           margin_of_safety: float
           signal_direction: str
           reasoning_trace: str
           sector_exposure: float = 0.0
           atr_multiplier: float = 3.0

       @dataclass
       class ApproveTradePlanCommand:
           symbol: str
           approved: bool
           modified_quantity: Optional[int] = None
           modified_stop_loss: Optional[float] = None

       @dataclass
       class ExecuteOrderCommand:
           symbol: str
       ```

    3. Create `src/execution/application/handlers.py`:
       - `TradePlanHandler` class accepting: `trade_plan_service: TradePlanService`, `trade_plan_repo: ITradePlanRepository`, `execution_adapter: AlpacaExecutionAdapter`.
       - All three via constructor DI.

       - `generate(cmd: GenerateTradePlanCommand) -> Optional[TradePlan]`:
         - Call `self._service.generate_plan(...)` with cmd fields.
         - If result is None (rejected by risk gate), return None.
         - Save plan to repo with status=PENDING.
         - Return the TradePlan.

       - `approve(cmd: ApproveTradePlanCommand) -> dict`:
         - Load plan from repo via `find_by_symbol(cmd.symbol)`.
         - If not found, return `{"error": "No pending plan for {symbol}"}`.
         - If cmd.approved is False, update status to REJECTED, return `{"status": "REJECTED", "symbol": cmd.symbol}`.
         - If cmd.modified_quantity or cmd.modified_stop_loss provided, update status to MODIFIED (create new TradePlan with modified values via dataclasses.replace if possible, or reconstruct). Save modified plan.
         - Otherwise update status to APPROVED.
         - Return `{"status": "APPROVED"|"MODIFIED", "symbol": cmd.symbol}`.

       - `execute(cmd: ExecuteOrderCommand) -> OrderResult`:
         - Load plan from repo via `find_by_symbol(cmd.symbol)`.
         - If not found or status not in (APPROVED, MODIFIED), raise ValueError.
         - Create BracketSpec from plan fields.
         - Call `self._adapter.submit_bracket_order(spec)`.
         - If result.status indicates success ("filled", "accepted"), update plan status to EXECUTED.
         - Else update to FAILED.
         - Return OrderResult.

    4. Add alert events to `src/execution/domain/events.py`:

       - `TargetReachedAlertEvent(DomainEvent)`: symbol, current_price, target_price
       These are simple event dataclasses. The monitoring command in Task 2 will check conditions and create these events.
