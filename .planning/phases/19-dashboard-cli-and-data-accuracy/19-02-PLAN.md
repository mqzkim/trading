Two targeted fixes in `src/dashboard/application/queries.py`:

**Fix 1: RiskQueryHandler drawdown (DASH-04/INT-02)**

In `RiskQueryHandler.__init__` (line 426-428), add `self._ctx = ctx` to store the full context dict (same pattern as OverviewQueryHandler and PipelineQueryHandler). Keep the existing `self._position_repo` and `self._regime_repo` extractions -- they remain used.

In `RiskQueryHandler.handle()` (lines 460-462), replace the hardcoded block:
```python
drawdown_pct = 0.0
drawdown_level = "normal"
```

With a portfolio query following the exact pattern from `OverviewQueryHandler._get_drawdown_pct()`:
```python
drawdown_pct = 0.0
drawdown_level = "normal"
handler = self._ctx.get("portfolio_handler")
if handler is not None:
    try:
        repo = handler._portfolio_repo
        portfolio = repo.find_by_id("default")
        if portfolio is not None:
            drawdown_pct = portfolio.drawdown * 100  # fraction -> percentage
            drawdown_level = portfolio.drawdown_level.value
    except Exception:
        pass
```

IMPORTANT: Multiply `portfolio.drawdown` by 100 because `build_drawdown_gauge()` expects percentage (0-20 range), but `Portfolio.drawdown` returns fraction (0.0-1.0). The overview page uses the raw fraction for its KPI card, but the risk gauge needs the percentage.

**Fix 2: Equity curve P&L accumulation (DASH-08/INT-03)**

In `OverviewQueryHandler._build_equity_curve()` (lines 273-296), replace the inner loop body that currently does nothing with actual P&L computation.

Current (broken):
```python
for trade in trades:
    date_str = str(trade.get("created_at", ""))[:10]
    dates.append(date_str)
    values.append(cumulative)  # always 0.0
```

Replace with:
```python
for trade in trades:
    date_str = str(trade.get("created_at", ""))[:10]
    entry = trade.get("entry_price", 0.0) or 0.0
    target = trade.get("take_profit_price", 0.0) or 0.0
    qty = trade.get("quantity", 0) or 0
    direction = trade.get("direction", "BUY")

    if entry > 0 and target > 0 and qty > 0:
        if direction == "BUY":
            trade_pnl = (target - entry) * qty
        else:
            trade_pnl = (entry - target) * qty
        cumulative += trade_pnl

    dates.append(date_str)
    values.append(round(cumulative, 2))
```

This uses the pragmatic approximation recommended in 19-RESEARCH.md: approximate realized P&L from entry/target price spread. This is an optimistic upper bound (assumes target hit), acceptable for v1 since the trade_plans table lacks exit_price.
