Implementation details:
1. Add the command at the end of the file (after all existing commands, before `if __name__`).
2. Use `@app.command()` decorator with Typer options for host, port, and no-browser flag.
3. Import `uvicorn` and `webbrowser` inside the function body (lazy import, matching existing patterns in the file).
4. Call `_get_ctx()` to bootstrap the context, then `create_dashboard_app(ctx)` to create the FastAPI app.
5. If `--no-browser` is not set, use `threading.Timer(1.5, lambda: webbrowser.open(url))` to schedule browser open before `uvicorn.run()` (which blocks).
6. Call `uvicorn.run(dashboard_app, host=_host, port=_port, log_level="info")`.
7. Fallback host/port from `src.settings.settings.DASHBOARD_HOST` / `DASHBOARD_PORT` when not provided via CLI.
8. Print startup message via `console.print()` before launching.
9. Wrap `webbrowser.open` in try/except to handle WSL2 edge cases gracefully.

The function signature:
```python
@app.command()
def serve(
    host: str = typer.Option(None, "--host", help="Bind host (default: from settings)"),
    port: int = typer.Option(None, "--port", help="Bind port (default: from settings)"),
    no_browser: bool = typer.Option(False, "--no-browser", help="Don't auto-open browser"),
):
```

Do NOT use subprocess to spawn uvicorn. Do NOT wrap in asyncio.run(). uvicorn.run() manages its own event loop.
