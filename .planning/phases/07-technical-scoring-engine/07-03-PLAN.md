**2. Fix `src/scoring/application/handlers.py` `_get_technical()` fallback (lines 192-196):**
The current fallback calls `compute_technical_score(symbol)` but the real signature is `compute_technical_score(df: DataFrame, indicators: dict)`. Replace the fallback with:
```python
def _get_technical(self, symbol: str) -> dict:
    if self._technical_client:
        return self._technical_client.get(symbol)
    # Fallback: fetch OHLCV, compute indicators, return raw values for sub-scoring
    from core.data.client import DataClient
    from core.data.indicators import compute_all
    from core.scoring.technical import compute_technical_score
    import math

    client = DataClient()
    df = client.get_price_history(symbol, days=756)
    ind = compute_all(df)
    result = compute_technical_score(df, ind)

    # Merge raw indicator values so _compute_technical_with_subscores() detects them
    def _last(s) -> float | None:
        v = s.dropna()
        if len(v) == 0:
            return None
        val = float(v.iloc[-1])
        return None if math.isnan(val) else val

    result["rsi"] = _last(ind["rsi14"])
    result["macd_histogram"] = _last(ind["macd_histogram"])
    result["close"] = float(df["close"].iloc[-1]) if len(df) > 0 else None
    result["ma50"] = _last(ind["ma50"])
    result["ma200"] = _last(ind["ma200"])
    result["adx"] = _last(ind["adx14"])

    # OBV change percentage (60-day lookback, same as TechnicalIndicatorAdapter)
    obv = ind["obv"].dropna()
    if len(obv) >= 60 and float(obv.iloc[-60]) != 0:
        result["obv_change_pct"] = (float(obv.iloc[-1]) - float(obv.iloc[-60])) / abs(float(obv.iloc[-60])) * 100
    else:
        result["obv_change_pct"] = None

    return result
```
This ensures that when the handler is called without an injected technical_client (which is the case in bootstrap -- see `ScoreSymbolHandler(score_repo=score_repo)` with no technical_client), it:
- Correctly calls `compute_technical_score(df, ind)` with proper arguments
- Returns raw indicator values that `_compute_technical_with_subscores()` can detect via `any(key in technical_data for key in ("rsi", "macd_histogram", "adx", "obv_change_pct"))`
- This triggers the TechnicalScoringService path which produces 5 sub-scores with explanations
