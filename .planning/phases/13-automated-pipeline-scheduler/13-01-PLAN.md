  <behavior>
    - SqlitePipelineRunRepository creates pipeline_runs table on init (run_id TEXT PK, started_at TEXT, finished_at TEXT, status TEXT, mode TEXT, stages_json TEXT, symbols_total INT, symbols_succeeded INT, symbols_failed INT, halt_reason TEXT, next_scheduled TEXT, error_message TEXT)
    - save() persists PipelineRun with stages serialized as JSON array
    - get_recent(limit) returns last N runs ordered by started_at DESC
    - get_by_id(run_id) returns PipelineRun or None
    - SQLite uses WAL mode (same pattern as SqliteCooldownRepository)
    - MarketCalendarService wraps exchange_calendars XNYS calendar
    - is_trading_day(date) returns True for NYSE sessions, False for weekends/holidays
    - is_early_close(date) returns True for early close sessions
    - next_trading_day(date) returns the next NYSE session date
    - Notifier protocol with notify(title, message, level) method
    - SlackNotifier sends webhook POST with emoji prefix, gracefully logs and continues if webhook URL is None or request fails
    - LogNotifier (fallback) just logs the notification via logger.info
  </behavior>
  <action>

    2. Create src/pipeline/infrastructure/market_calendar.py with MarketCalendarService. Create exchange_calendars.get_calendar("XNYS") once in __init__. is_trading_day uses _cal.is_session(pd.Timestamp(d)). is_early_close checks pd.Timestamp(d) in _cal.early_closes. next_trading_day uses _cal.date_to_session(pd.Timestamp(d), direction="next").
