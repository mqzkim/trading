  <behavior>
    - PipelineOrchestrator.run(handlers, symbols, dry_run, mode) returns PipelineRun
    - Runs 6 stages sequentially: ingest, regime, score, signal, plan, execute
    - Each stage wrapped in _retry_stage() with 3 attempts, exponential backoff (1s, 2s, 4s)
    - Ingest stage: calls DataPipeline.ingest_universe() via asyncio.run(), collects succeeded_symbols
    - Regime stage: calls regime_handler.handle() with DetectRegimeCommand, extracts RegimeType
    - Score stage: calls score_handler.handle() for each surviving symbol, collects per-symbol scores, skips failed symbols
    - Signal stage: calls signal_handler.handle() for each scored symbol, collects signals, skips failed symbols
    - Plan stage: calls trade_plan_handler with signal results to generate trade plans
    - Execute stage: submits trade plans via trade_plan_handler, skipped if dry_run=True
    - Before execute, _should_halt() checks: RegimeType.CRISIS -> halt, DrawdownLevel in (WARNING, CRITICAL) -> halt
    - Halt returns PipelineRun with status=HALTED and halt_reason describing regime/drawdown
    - Partial ingest failure: continue with succeeded symbols only
    - Per-symbol score/signal failure: skip symbol, log error, continue with rest
    - RunPipelineCommand(dry_run: bool, mode: RunMode)
    - RunPipelineHandler orchestrates: call reconciliation_service.check_and_halt() first, then create PipelineRun, call orchestrator.run(), save to repo, notify, return result. ReconciliationError aborts entire run with FAILED status.
    - GetPipelineStatusQuery(limit: int = 10)
    - PipelineStatusHandler queries repo for recent runs
  </behavior>
  <action>

This is a pure domain service -- receives handler references via constructor or run() params, never imports from other bounded contexts directly. The orchestrator does NOT call reconciliation directly (that is the application layer's responsibility per DDD). Key methods:
       - run(handlers: dict, symbols: list[str], dry_run: bool = False, mode: RunMode = RunMode.MANUAL) -> PipelineRun
       - _run_stage(name: str, fn: callable) -> StageResult (wraps fn with timing and error capture)
       - _retry_stage(fn: callable, max_retries: int = 3, base_delay: float = 1.0) -> any (exponential backoff loop)
       - _should_halt(regime_type: RegimeType, drawdown_level: DrawdownLevel) -> bool (Crisis or WARNING/CRITICAL)

       - Regime: `handlers["regime_handler"].handle(DetectRegimeCommand())` -- returns regime result. Extract regime_type.
       - Score: Loop over surviving symbols, call `handlers["score_handler"].handle(ScoreSymbolCommand(symbol=s))`. Catch per-symbol exceptions, log and skip.
       - Signal: Loop over scored symbols, call `handlers["signal_handler"].handle(GenerateSignalCommand(symbol=s))`. Catch per-symbol exceptions, log and skip.
       - Plan: Call `handlers["trade_plan_handler"]` with signal results to generate trade plans. Details depend on existing handler API -- read the handler to understand the command.
       - Execute: Submit generated plans via trade_plan_handler. Skip entirely if dry_run=True or _should_halt() returns True.
