<task type="auto">
  <name>Task 1: Create JSON API router and mount on FastAPI app</name>
  <files>src/dashboard/presentation/api_routes.py, src/dashboard/presentation/app.py</files>
  <action>
Create `src/dashboard/presentation/api_routes.py` with `APIRouter(prefix="/api/v1/dashboard", tags=["dashboard-json"])`.

**GET endpoints (4):**
1. `GET /overview` -- Call `OverviewQueryHandler(ctx).handle()`, strip `equity_curve_chart_json` key if present (per RESEARCH.md). Return dict directly (FastAPI auto-serializes).
2. `GET /signals` -- Call `SignalsQueryHandler(ctx).handle(sort_by=sort, sort_desc=desc)`. Accept `sort: str = "composite"` and `desc: bool = True` query params.
3. `GET /risk` -- Call `RiskQueryHandler(ctx).handle()`, strip `gauge_json` and `donut_json` keys from the returned dict (Plotly data not needed by React frontend). Return remaining dict.
4. `GET /pipeline` -- Call `PipelineQueryHandler(ctx).handle()`, return dict directly.

All GET endpoints: `ctx = request.app.state.ctx`, instantiate handler with ctx, call handle(), return data.

**POST endpoints (5) -- accept Pydantic BaseModel JSON bodies (NOT Form data):**
1. `POST /pipeline/run` -- Body: `PipelineRunRequest(symbols: list[str] = [], dry_run: bool = False)`. Same background-thread logic as existing HTMX route in routes.py: parse symbols, create RunPipelineCommand, execute in daemon thread. Return `{"status": "running", "symbols": [...], "dry_run": bool}`.
2. `POST /approval/create` -- Body: `ApprovalCreateRequest(score_threshold: float = 70.0, allowed_regimes: list[str] = ["Bull", "Accumulation"], max_per_trade_pct: float = 8.0, daily_budget_cap: float = 10000.0, expires_in_days: int = 30)`. Call `ctx["approval_handler"].create(CreateApprovalCommand(...))`. Return refreshed approval status via `PipelineQueryHandler`.
3. `POST /approval/suspend` -- Body: `ApprovalActionRequest(approval_id: str | None = None)`. Call `ctx["approval_handler"].revoke(RevokeApprovalCommand(...))`. Return refreshed approval status.
4. `POST /approval/resume` -- Body: `ApprovalActionRequest(approval_id: str | None = None)`. Call `ctx["approval_handler"].resume(ResumeApprovalCommand(...))`. Return refreshed approval status.
5. `POST /review/approve` -- Body: `ReviewActionRequest(review_id: int)`. Call `ctx["review_queue_repo"].mark_reviewed(body.review_id, approved=True)`. Return refreshed review queue.
6. `POST /review/reject` -- Body: `ReviewActionRequest(review_id: int)`. Call `ctx["review_queue_repo"].mark_reviewed(body.review_id, approved=False)`. Return refreshed review queue.

**SSE endpoint (1):**
`GET /events` -- Async endpoint. Get `bridge = request.app.state.sse_bridge`. Create async generator that iterates `bridge.stream()`, checks `request.is_disconnected()`, and yields `ServerSentEvent(data=json.dumps(payload), event=event_type)`. Return `EventSourceResponse(event_generator())`. This sends raw JSON payloads (no HTML partials, unlike the HTMX SSE endpoint).

**Pydantic models** (define at top of api_routes.py):
- `PipelineRunRequest`, `ApprovalCreateRequest`, `ApprovalActionRequest`, `ReviewActionRequest`

**Mount in app.py:**
Add `from src.dashboard.presentation.api_routes import api_router` and `app.include_router(api_router)` after the existing `app.include_router(router)` line in `create_dashboard_app()`.

**Important:**
- Import patterns from existing routes.py for POST endpoint logic (RunPipelineCommand, CreateApprovalCommand, etc.)
- Do NOT modify queries.py or routes.py -- keep existing HTMX routes unchanged
- Use `from __future__ import annotations` at top of api_routes.py (project convention)
