    - POST /api/v1/auth/keys with valid JWT creates API key, returns key_id + raw_key (shown once)
    - GET /api/v1/auth/keys with valid JWT lists user's keys (no raw key material)
    - DELETE /api/v1/auth/keys/{key_id} with valid JWT revokes the key
    - DELETE /api/v1/auth/keys/{key_id} for wrong user returns 404
    - Any protected endpoint without Authorization header returns 401
    - Any protected endpoint with expired JWT returns 401

  <action>
    1. Create `commercial/api/routers/auth.py`:
       - `router = APIRouter(prefix="/auth", tags=["Authentication"])`
       - `POST /token`: Accept `x-api-key` header, verify via ApiKeyRepository.verify_key(), generate JWT with PyJWT containing claims {sub: user_id, tier: tier, exp: now + TOKEN_EXPIRE_HOURS}. Return TokenResponse.
       - `POST /keys`: Requires valid JWT (Depends(get_current_user)). Accept APIKeyCreate body (name field). Create key via ApiKeyRepository, return APIKeyResponse with key_id and raw_key.
       - `GET /keys`: Requires JWT. List keys via ApiKeyRepository.list_keys(user_id). Return APIKeyListResponse.
       - `DELETE /keys/{key_id}`: Requires JWT. Revoke via ApiKeyRepository.revoke_key(key_id, user_id). Return 204 on success, 404 if key not found or not owned by user.
